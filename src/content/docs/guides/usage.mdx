## Recommended Project Structure

Tungsten works best when your raw assets are kept outside of your active source code. This ensures your project only references the generated Luau files.

<FileTree>
- assets/
  - packed/ # Icons and UI elements
  - individual/ # Backgrounds and unique images
- src/
  - shared/
    - Assets.luau # Generated by Tungsten
- tungsten.toml
- tungsten.lock.toml
</FileTree>

## The Development Loop

<Steps>

1.  ### Modify
    Add or update files in your `assets/` directory.

2.  ### Verify
    Run:
    ```bash title="Terminal"
    tungsten test
    ```
    to check your glob patterns and config validity.

3.  ### Sync
    Run:
    ```bash title="Terminal"
    tungsten sync --target roblox
    ```
    to upload changes and update your Luau files.

</Steps>

:::tip
  Use `tungsten sync --target none` to perform a "dry run." This shows you exactly what *would* be uploaded without touching your Roblox inventory or API quota.
:::
