Tungsten isn't just a syncing tool; it includes a powerful processing pipeline that allows you to manage raw source files (like SVGs) while delivering optimized formats (like PNGs) to Roblox.

## Image Transformation

Tungsten can transform images on the fly based on rules defined in your `tungsten.toml`. This is handled entirely locally, prioritizing your privacy and ensuring no data is sent to external APIs.

### Format Conversion
The `convert` field allows you to define rules for changing file extensions during the sync process. Rules follow a `source -> target` format.

- **Global Rules:** `"svg -> png"` converts every SVG matched by the input glob into a PNG.
- **Specific File Rules:** `"hero.jpg -> hero.png"` targets a single file by name.
- **Path-Specific Rules:** `"assets/ui/icon.png -> assets/ui/icon.webp"` targets a specific file path.

### SVG Rasterization
Since Roblox does not natively support vector graphics, Tungsten rasterizes SVGs into high-quality PNGs. You can control the output resolution using the `svg_scale` field.

```toml title="tungsten.toml"
[inputs.icons]
path = "assets/icons/**/*.svg"
output_path = "src/Icons.luau"
svg_scale = 2.0  # Rasterize at 2x the native resolution.
# Tungsten automatically converts SVGs to PNGs
# but you can specify which file format you want it to convert to.
convert = ["svg -> jpg"]
```

## Automatic Spritesheet Packing

One of Tungsten's core features is the ability to pack multiple images into a single spritesheet. This significantly reduces draw calls and improves memory efficiency in your Roblox game.
Enabling Packing

To enable this, set `packable = true` within an input block.
```toml title="tungsten.toml
[inputs.packed_ui]
path = "assets/ui/*.png"
output_path = "src/UI.luau"
packable = true
```

### How it Works

<Steps>

1.  #### Discovery
    Tungsten finds all images matching your glob pattern.

2.  #### Bin Packing
    It uses an optimized algorithm to fit these images into the smallest possible square spritesheet.

3.  #### In-Memory Processing
    The spritesheet is generated in your computer's memory. It is never saved to your local disk, keeping your workspace clean.

4.  #### Upload & Reference
    Tungsten uploads the sheet and generates Luau code that includes the ImageRectOffset and ImageRectSize for each sub-image.

</Steps>
